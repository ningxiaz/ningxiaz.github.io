UI/UX@MOBIATAUI/UX for mobile web offline itinerary of Expedia

In the summer of 2012, I interned at Mobiata, which is a startup acquired by Expedia, as a Mobile Designer. One of the intern projects I did is offline itinerary on mobile web. The point of view is, the travelers need an easy way to access their itineraries on the way, even if the internet availability is low and unreliable, which is often the case during travel. With the local storage feature of the web browser, it’s possible to store the itinerary information in the web browser for offline usage. My responsibility was to make the feature visible and the experience smooth and enjoyable.

In order to enable fast access, we encourage users to save the web page on the home screen. Thus we need a non-intrusive notice to acknowledge users. For Safari browsers on iOS, we can drop a bubble labeled “Add to home screen for offline viewing”, with an arrow pointing downwards to the Add button. For other browsers and Android devices, there is no universal way to save a web page to home scree, thus a general notice is designed, with appropriate styles that fit the other visual elements and Android look and feel. For better UX, those notices shouldn’t appear when users don’t need them. Thus, they disappear after certain amount of time, and won’t appear again if the user clicks to dismiss.

For the home screen icons, I have explored more than 6 concepts before arriving the luggage tag to represent the idea of itinerary. The other icon is just for the whole Expedia web app. Both icons were crafted to fit in the iOS 7 visual style, while retaining the branding and clarity. Similar icons were created for Android and other devices.

Then it comes to the actual use scenarios of offline itinerary. When the user is connected, a brief message is shown to indicate the information has saved, and is up to date. When the user is offline, the user needs to be informed that 1) the app is not connected to the Internet now 2) the information might be outdated and needs to be refreshed when connection is available. To convey these messages in a minimal and unambiguous way, we’ve brainstormed several solutions, and ended up having a message bar stating the offline status, together with the elapsed time since the last update. The green and yellow color dots in both cases are also helpful in understanding the status.

More than 10 directions of visual styles were explored for the message bar.

The major part of my internship was to work on the ideation of the next tablet app by Mobiata. Since it’s still under development, it’s now shown here.
